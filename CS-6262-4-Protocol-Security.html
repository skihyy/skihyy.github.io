CS 6262 - Network Security - Protocol Security

  • Circuit and packet switching

    • Circuit switch
      • Must establish route before communication.
      • Once start, route will not end before communication is over.
      • Single route.
    • Package switching
      • Internet
      • Data split into packets.
      • Packets transported independently through network.
      • Each packet handled on a best efforts basis.
      • Packets may follow different routes.

  • Protocols

    • Rules of communications.
    • Connectionless
      • Information is set up in each data packets, instead of being pre-defined in the first packet.
      • UDP (user datagram protocol)
    • Connection-oriented
      • Establish communication session before transmitting data.
      • A connection stream between two nodes.
      • 3 phases - set up, transmission, tear down.
      • Create virtual circuit-switch network.
      • TCP (transmission control protocol)

  • Encapsulation

    • A packet has header, footer, and playload.
      • Header, footer - control info and addressing.
      • Payload: data
    • A packet P1 of network protocol N1 is encapsulated into a packet P2 of another protocol N2.
      • Payload of P2: N1.
      • >

  • Network Layers

    • Stack of layers
      • Higher layer can be encapsulation in lower layers.
      • A layer can be implemented in hardware or software.
      • The bottommost layer must be in hardware.
    • A communication between two nodes is established for each layer.
      • Actual channel at the bottom layer.
      • Virtual channel at the higher layer.

    • Internet layers.

      • Intermediate Layers
        • Link Layer
          • Local area: Ethernet, Wifi, optical fiber.
          • 48-bit MAC (media access control) address.
          • A packet called frame.
        • Network Layer
          • Internet-wide communication
          • Best efforts
          • 32-bit IPv4, 128-bit IPv6
        • Transport Layer
          • 16-bit address (port)
          • TCP & UDP

  • OSI (Open System Interconnect) Model - 7 layers

  • MAC address

    • 48-bit in hex, with first 3 pre-defined octets.
    • The last 3 must be unique.

  • Switch

    • Link layer.
    • Learn MAC address of every connected computer.
    • Combine multiple switches as a tree.
    • Fragments to unknown MAC are broadcast.

  • Address Resolution Protocol (ARP)

    • Connect network layer to data layer by converting IP to MAC.
    • Broadcasting requests and caching response for future use.
    • Request: who has <IP 1> tell <IP 2>
    • When machine with <IP 1> or an APR server receives the message, it sends response to`.
    • Spoofing
      • ARP table is updated whenever an ARP response is received.
      • Requests are not tracked.
      • ARP announcements are not authenticated.
      • Machines trust each other.
    • Poisoning
      • ARP - almost stateless
      • Update ARP table when receiving response, even it does not send request.
      • Poisoning by sending responses.
      • Using static entries can solve but impossible to manage.

  • IP routing

    • A router bridges 2 or more networks.
      • At network layer.
      • Maintain table to forward packets.
      • Forward decisions based solely on the destination address.
    • Routing table
      • Addresses to LANs or other gateway routers.

  • Internet Routes

    • Internet Control Message Protocol (ICMP)
      • For network test and debug.
      • Simple message encapsulated in single IP packets.
      • Network layer protocol.
    • Tools on ICMP
      • Ping
      • Traceroute: send series ICMP packets with increasing TTL to discover routes
    • ICMP attack
      • Too long packets (> 64KB) will be fragmented which cause systems to crash.

  • IP Vulnerabilities

    • Unencrypted transmission - can be listened at any intermediate host during routing.
    • No source authentication - difficult to track back to spoof makers.
    • No integrity checking - content forgeries, redirections, and man-in-the-middle attack.
    • No bandwidth constraints - denial-of-service attack.

  • TCP (transmission control protocol)

    • Stream of 8-bit byte data, packages into a segment.
    • Delivery order maintained by making each packet a sequence number.
    • Send ACK after successful sending.
    • Check data by comparing a checksum encoded in the packet.

  • Ports

    • Support multiple concurrent applications on the same layer.
    • 16-bit numbers.
    • TCP header has ports of sender and receiver.

  • TCP connection establishing

    • 3-way hand shake.
      • Client sends SYN packet with seq = x.
      • Server responds SYN/ACK with seq = y and ack = x + 1.
      • Client responds ACK with seq = x + 1 and ack = y + 1.
    • SYN flood
      • DDoS - large number of packets with spoofed source.
      • Server responds SYN/ACK and wait for 3 minutes.
      • Common solution is use SYN cookies (server will not store connection until SYN/ACK is verified).
    • Optimistic ACK attack
      • Sending ACK when not received.
      • Server stack believe there is large amount bandwidth available and increase cwnd.
      • No practical solution.
    • TCP session hijacking
      • Sniff victim's packets (ARP poisoning, or man-in-the-middle attack)
      • Read seq and ack, and synchronize attacker's TCP/IP stack.
      • Send RST to client (or DoS client).
      • Craft request to server, receive responses. Port knocking
      • Making connections to blocked ports in an attempt to open a port.
      • Fairly secure against brute force attack (65536^k combinations, k is the number of ports knocked).
      • Real work - susceptible. One can record the track and re-open the port.

Internet inter-AS routing - BGP

  • BGP (Border Gateway Protocol)

    • Provide each AS (automous system) a means to:
      • Obtain subnet reachability info from neighboring ASs.
      • Propaganda reachability info to all AS-internal routers.
      • Determine "good" routers to subnets based on reachability info and policy.
    • Allows subnet to advertise its existence.
    • Pairs of routers (BGP peers) exchange routing info over simi-permanent TCP connections (BGP sessions - no nned to correspond to physical links)
    • When AS2 advertises a prefix to AS1, AS2 promises it will forward all datagrams destinated to that prefix.

  • Path attributes & BGP routes

    • Advertised predix includes BGP attributes
      • prefix + attributes = "route"
    • Attributes x2:
      • AS_PATH: contains ASs through which prefix advertisement has passed (e.g. AS 67, AS 17) (loops avoid by checking AS list)
      • NEXT-HOP: interface which begins AS-PATH. Indicates specific router interface through which packets can be routed (e.g. 1d learns the IP of interface of 2a and identifies its forwarding interface).
      • Important policy to accept / decline announcements.

  • BGP route selection

    • Router may learn more than 1 route to the same prefix, but MUST SLECT ONE.
    • Simplified elimination rules:
      • Local preference value attribute: policy decision.
      • Shortest AS-PATH.
      • Closest NEXT-HOP router.
      • Additional criteria.

  • BGP messages (TCP with port 127)

    • OPEN: open TCP connection to peer and authenticates sender.
    • UPDATE: advertise new path.
    • KEEPALIVE: keep connection alive in absense of UPDATES, ALSO `ACKs OPEN request.
    • NOTIFICATION: reports errors in the previous message, also used to close connection.
    • Example:
      • A advertises to B the path AW.
      • B advertises to X the path BAW.
      • Should B advertise to C the path BAW?
        • Does B get revenue?
        • If no:
          • B wants to force C to route to W via A.
          • B wants to route only to / from its customers.

  • Difference of Intra- and Inter-AS routing

    • Intra-AS: single admin, no policy decision needed.
    • Inter-AS: admin wants to control over how its traffic routed, who routes through its net.
    • Scale: hierarchical routing saves table size, reduced update traffic.
    • Performance
      • Intra-AS: focus on performance.
      • Inter-AS: policy may dominate over performance.
